# User Story 007.2: Implement Diary Entry Form - Part 2 (Selects, Chips, and Submission)

## 1. Title

Implement select fields, multi-select chip components, and enhance form submission for the diary entry form.

## 2. Goal

To complete the diary entry form by implementing the select fields ('Location', 'Company', 'Entry Type'), multi-select chip components ('Emotions', 'Behavior'), 'Skipped Meal' toggle, and enhance the form submission process.

## 3. Description

As a junior developer, I need to add the remaining interactive form elements to the diary entry form and improve the form submission handling. This includes implementing select fields, multi-select chip components, a toggle switch, and setting up basic form data handling for all fields.

## 4. Technical Details

- **MUI Components:**
  - `Select` and `MenuItem` for 'Location', 'Company', 'Entry Type'.
  - `Autocomplete` with `multiple` prop for 'Emotions' and 'Behavior' (multi-select chips).
  - `Switch` and `FormControlLabel` for 'Skipped Meal' toggle.
- **State Management:** Use `useState` for all new fields ('emotions', 'location', 'company', 'behavior', 'skippedMeal', 'entryType').
- **Form Submission:** Update `handleSubmit` function to include all form field values in the `console.log` output.

## 5. Steps to Implement

1. **Open `EntryForm.tsx`:**
   - Open the `src/components/EntryForm.tsx` file created in User Story 007.1.
2. **Import Additional MUI Components:**
   - Import `Select`, `MenuItem`, `Switch`, `FormControlLabel`, `Autocomplete`, `Chip` from '@mui/material'.
3. **Implement State Variables for New Fields:**
   - Use `useState` to create state variables for:
     - `emotions`: Array of strings, initially empty array `[]`.
     - `location`: String, initially empty string `''`.
     - `company`: String, initially empty string `''`.
     - `behavior`: Array of strings, initially empty array `[]`.
     - `skippedMeal`: Boolean, initially `false`.
     - `entryType`: String, initially empty string `''`.
4. **Implement Select Fields:**
   - Add MUI `Select` components for 'Location', 'Company', and 'Entry Type' within the `EntryForm` component.
   - Use `MenuItem` components for the options in each `Select` field (define options as needed, e.g., for 'Location': Home, Work, Restaurant, etc.).
   - Connect each `Select` to its corresponding state variable using `value` and `onChange` props.
5. **Implement Multi-Select Chip Components (Autocomplete):**
   - Add MUI `Autocomplete` components with `multiple` prop for 'Emotions' and 'Behavior'.
   - Use `Chip` components to render selected values in the Autocomplete.
   - Define options for 'Emotions' and 'Behavior' (e.g., arrays of strings).
   - Connect each `Autocomplete` to its corresponding state variable using `value` and `onChange` props.
6. **Implement 'Skipped Meal' Toggle Switch:**
   - Add MUI `Switch` component wrapped in `FormControlLabel` for 'Skipped Meal'.
   - Connect the `Switch` to the `skippedMeal` state variable using `checked` and `onChange` props.
7. **Update `handleSubmit` Function:**
   - Modify the `handleSubmit` function to log all form field values to the console when the form is submitted. Include `emotions`, `location`, `company`, `behavior`, `skippedMeal`, and `entryType` in the `console.log` output.
8. **Verify in Browser:**
   - Run `npm run dev` and navigate to `/entry/create`.
   - You should see the complete diary entry form with all fields, including selects, chips, and the toggle switch.
   - Fill in all form fields and submit the form.
   - Check the browser console to see the logged form data, ensuring all field values are correctly captured.

## 6. Acceptance Criteria

- MUI `Select`, `MenuItem`, `Autocomplete`, `Chip`, `Switch`, `FormControlLabel` components are used for the respective form fields in `EntryForm.tsx`.
- State management using `useState` is implemented for 'emotions', 'location', 'company', 'behavior', 'skippedMeal', and 'entryType' fields.
- All form fields are rendered in the `EntryForm` component and are visually complete in the browser.
- Form submission captures and logs all field values to the console, including selects, chips, and toggle switch values.
